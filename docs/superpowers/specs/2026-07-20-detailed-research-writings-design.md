# Detailed Research Writings Design

## 목적

두 개의 Writing 글을 논문 요약이 아니라 연구 과정을 따라갈 수 있는 상세 케이스 스터디로 확장한다. 독자는 문제 설정, 데이터, 방법 선택, 실험 설계, 결과와 한계를 순서대로 이해할 수 있어야 한다. 보고서와 논문에 없는 해석이나 역할은 덧붙이지 않는다.

이 문서는 기존 음성 비식별화 설계를 대체하며, 수치는 제공된 위탁과제 보고서와 게재 논문을 우선 기준으로 삼는다.

## 공통 편집 원칙

- 연구자가 실제로 내린 선택과 그 근거를 중심으로 쓴다.
- 본문 종결어미는 논문체 `~했다`, `~다`로 통일한다. `~했습니다`, `~입니다`는 사용하지 않는다.
- “의미가 있다”, “효과적이다” 같은 추상적인 결론은 구체적인 수치나 관찰로 바꾼다.
- 전문용어는 처음 등장할 때만 풀어서 설명하고 이후에는 약어를 사용한다.
- 긴 문단을 피하고 표, 결과 카드, 그림 설명으로 정보 밀도를 분산한다.
- 논문과 보고서의 결과가 다른 경우 자료의 평가 조건을 함께 적고 임의로 합치지 않는다.
- 원본 통화음성, 대화 내용, 화자 식별 정보, 참가자 개인정보는 공개하지 않는다.

## 음성 비식별화 글

### 추가할 내용

1. 원천 데이터 구성을 기관별로 제시한다.
   - 창원시: 78,114개, 81.93 GB
   - 한국우편사업진흥원: 10,430개, 27.95 GB
   - 한국소비자원: 8,741개, 70.1 GB
   - 합계: 97,285개, 약 180 GB, 9개 상담 도메인
2. AI Hub 화자 인식용 한국어 음성에서 후보 기법을 선별한 단계를 분리한다.
   - 전체 코퍼스 3,000명·7,000시간
   - 100명 × 30개 발화, 총 3,000개 표본
   - ECAPA-TDNN EER과 직접 청취를 함께 사용
   - Modspec은 음질 저하로 제외하고 McAdams, Resampling, VTLN을 선택
3. JSON 라벨의 화자 ID와 시작·종료 시점을 사용한 단일 화자 전처리를 구체화한다.
4. 청취평가와 자동평가를 별도 절차로 설명한다.
   - 10대부터 50대까지 연령대별 10명, 총 50명
   - 5개 테스트 세트, 세트당 300문항, 총 1,500문항
   - 142명 화자, 중복 포함 3,000개 음성
   - 자동평가 1,677개 음성: 남성 837개, 여성 840개
5. 유용성 평가의 시행착오를 숨기지 않는다.
   - CLOVA Speech Recognition 결과는 원본에서도 CER이 높아 평가 기준으로 쓰기 어려웠음
   - 사람 받아쓰기에서는 변조 방법별 CER이 20% 미만이었음
6. 사람 평가와 자동평가의 순위가 달랐다는 점을 명확히 한다.
   - 사람 평가: VTLN 유사도 1.52로 최저
   - 자동평가: Pitch 43.68%, VTLN 44.00%, Resampling 46.39% EER

### 시각자료

- 기존 평가 흐름도, 기법 비교 그래프, 청취평가 화면, McAdams/VTLN 논문 그림을 유지한다.
- 기관별 데이터 개수와 용량을 보여 주는 원본 SVG 차트를 추가한다.
- 프라이버시와 유용성 평가를 함께 읽을 수 있는 요약 패널을 HTML로 추가한다.
- 참가자 모집 화면이나 개인정보 관련 양식은 사용하지 않는다.

## 생성 음성 탐지 글

### 추가할 내용

1. 기존 스펙트럼·파형 기반 탐지와 Encodec-BERT 표현의 차이를 짧게 비교한다.
2. 특징 추출 과정을 단계별로 설명한다.
   - Encodec 인코더와 residual vector quantization
   - 두 번째 양자화 레이어 출력 사용
   - `[2, 512]` 형태로 정리한 이산 코드
   - BERT-Base: 12개 인코더 레이어, 768차원 hidden state, 12개 attention head
3. 데이터 표를 bona fide와 spoof의 개별 수량까지 확장한다. 각 분할은 1:1로 맞춘 값임을 명시한다.
4. 실험을 두 단계로 나눈다.
   - 1단계: ASVspoof 2019 LA에서 FC-layer, Transformer, ResNet-18, ResNet2와 비교 모델을 평가
   - 2단계: 상위 세 분류기를 ASVspoof 2021 LA와 DF에서 평가
5. 전체 정확도뿐 아니라 클래스별 정확도, Balanced Accuracy, EER을 함께 읽도록 설명한다.
6. LA에서는 FC-layer, DF에서는 ResNet2가 가장 낮은 EER을 보인 결과와 클래스별 성능 차이를 한계로 기록한다.

### 시각자료

- Encodec-BERT 구조, 분류기 평가 흐름, LA·DF 결과 그림을 유지한다.
- 2019 LA에서 2021 LA·DF로 이어지는 2단계 실험 설계를 원본 SVG로 추가한다.
- 핵심 결과는 HTML 카드와 표에도 제공해 논문 그림의 작은 글씨에만 의존하지 않게 한다.

## 화면 구성

- 두 글 모두 `연구 개요 → 문제 → 데이터 → 방법 → 평가 → 결과 → 한계 → 논문` 순서를 따른다.
- 큰 섹션 사이에 작은 영문 장 번호를 두어 긴 페이지에서 현재 위치를 알 수 있게 한다.
- `자료 구성`, `실험 설계`, `핵심 관찰`, `평가상의 한계`를 위한 공통 callout 스타일을 추가한다.
- 기존 흑백 중심 색상과 Pretendard 본문을 유지하고, 강조색은 선과 작은 라벨에만 사용한다.
- 한국어 제목과 본문은 단어 내부에서 끊지 않는다. 제목은 균형 줄바꿈, 본문은 자연스러운 문장 줄바꿈을 사용한다.
- 작은 화면에서 Writing 목록의 기간과 제목은 한 열로 배치해 제목 폭을 확보한다.
- 모바일에서는 표를 가로 스크롤하고, 그림 쌍과 결과 카드를 한 열로 배치한다.

## 검증 기준

- 제공된 보고서·논문의 데이터 수량, 기간, 모델 구조와 결과 수치가 글과 표에서 일치한다.
- 모든 그림에는 내용을 설명하는 대체 텍스트와 출처가 드러나는 캡션이 있다.
- 생성된 두 HTML 페이지에 깨진 Markdown 표기나 Liquid/Nunjucks 문법이 남지 않는다.
- 두 Markdown 원고에 `했습니다`, `입니다`, `됩니다`, `있습니다` 종결어미가 남지 않는다.
- `npm run build`와 `git diff --check`가 성공한다.
- 모바일 폭에서도 텍스트, 표, 그림 캡션이 겹치지 않는다.
